# F-002 — The manager emits a report after verification retry exhaustion

**Evidence level:** `synthetic-orchestration`  
**Severity:** medium within the harness boundary; not a security classification

## Observed RED

In FR-006, the deterministic verifier rejects the initial report and both permitted revisions. The
frozen manager exits the loop, prints the still-unverified report, and prints the negative verifier
object. The frozen oracle requires a controlled stop before terminal report emission.

- Baseline: `baseline-002`, FR-006 = FAIL
- Observed: `verificationCalls=3`, `revisionCalls=2`, `terminalReportEmitted=true`
- Evidence: [`../evidence/runs/baseline-002/results.json`](../evidence/runs/baseline-002/results.json)
- Reproduce RED: `npm run teardown:baseline`

## Root cause

The loop correctly enforces two revisions, but terminal report output is unconditional after the loop.
There is no branch for a final `verification.verified === false` value.

## Remediation and GREEN

The patch adds one post-loop guard that throws a descriptive error containing the revision count and
final verifier issues. The unchanged corpus then observes three verification calls, two revisions, a
controlled error, and no terminal report.

- Remediation: `remediation-001`, FR-006 = PASS
- Patch: [`../patches/openai-agents-js-financial-research-fail-closed.patch`](../patches/openai-agents-js-financial-research-fail-closed.patch)
- Reproduce GREEN: `npm run teardown:verify-fix`

## Claim boundary

This proves only the frozen manager's terminal decision with deterministic verifier outputs. It does
not validate verifier correctness, establish user harm, or measure production incidence.
