# F-001 — The writer runs after every search fails

**Evidence level:** `synthetic-orchestration`  
**Severity:** medium within the harness boundary; not a security classification

## Observed RED

In FR-003, two deterministic search boundaries return failure. The frozen manager's real
`performSearches()` method returns an empty array, but `run()` calls the writer once and emits a
terminal report. The frozen oracle requires a controlled stop before the writer.

- Baseline: `baseline-002`, FR-003 = FAIL
- Observed: `outcome=success`, `writerCalls=1`, `terminalReportEmitted=true`
- Evidence: [`../evidence/runs/baseline-002/results.json`](../evidence/runs/baseline-002/results.json)
- Reproduce RED: `npm run teardown:baseline` (compares a fresh ignored run to the recorded baseline)

## Root cause

`search()` converts all search-agent exceptions to `null`, and `performSearches()` removes those nulls.
`run()` passes the resulting array directly to `writeReport()` without checking that any usable source
summary remains.

The two halves rest on different evidence, and the distinction matters. The aggregation and the
unguarded call into `writeReport()` (`manager.ts:46-49`) are `synthetic-orchestration`: that code
runs in every corpus case. The exception-to-null conversion (`manager.ts:100-107`) is
`static-analysis` only — the harness overrides `search()` and returns `null` directly, so no corpus
case executes that `catch` block. What was measured is that a null-only result set reaches the
writer; that exceptions are what produce the nulls in production is read from the source, not
observed here.

## Remediation and GREEN

The patch adds one guard after aggregation: if zero usable results remain, throw a descriptive error
before calling the writer. The unchanged corpus then observes `writerCalls=0` and no terminal report.

- Remediation: `remediation-001`, FR-003 = PASS
- Patch: [`../patches/openai-agents-js-financial-research-fail-closed.patch`](../patches/openai-agents-js-financial-research-fail-closed.patch)
- Reproduce GREEN: `npm run teardown:verify-fix`

## Claim boundary

This proves only the frozen manager control flow with deterministic agent boundaries. It does not
show that a real model fabricates facts, estimate production frequency, or evaluate financial quality.
